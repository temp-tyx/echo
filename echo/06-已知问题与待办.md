## 待排查方向

### A. `_prepare_input_ids` 索引计算

当前逻辑：

```python
flat_end = int(cu_num_tokens[cur_index]) - 1
draft_len = len(spec_tokens)
sample_idx = flat_end - draft_len
spec_start = sample_idx + 1
```

需验证 `cu_num_tokens` 与 ECHO trim 后的 `num_scheduled_tokens` 是否一致；多 req varlen batch 下 `sample_idx` 是否指向正确 slot。

### B. `prev_sampled_token_ids` 写入时机

async scheduling 下 `prev_positions.np[cur_index]` 可能为 `-1` 或指向错误行，导致 sample 位未写入 → 后续 draft 链断裂。

### C. req2 `spec_num_draft=0`

完全剪枝的 req 仍参与 batch forward 时，`logits_indices` / `num_draft_tokens` 是否与 scheduler 一致需对照 `[target_prepare]` 日志。

### D. positions / slot_mapping

虽已从 req 错配方向修复，仍需确认 ECHO varlen batch 下 attention metadata 与 token 布局对齐（对照 `[target_batch]` 的 `uniform_decode` 与 padding）。
