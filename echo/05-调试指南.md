# 05 — 调试指南

[← 返回目录](./README.md)

## 开启调试

```bash
export VLLM_ECHO_DEBUG=1
python your_script.py 2>&1 | grep "ECHO \["
```

## 日志阶段

| 前缀 | 位置 | 关键字段 |
|------|------|----------|
| `[draft_propose]` | propose 前 | `echo_k_max`, `batch_size`, `draft_step`, `req_ids` |
| `[prune]` | Eagle 剪枝 | `cum_log_probs`, `mask`, `kept`, `top_cells`, `draft_before/after` |
| `[draft_result]` | propose 后 | `echo_cu_draft_tokens`, `echo_draft_tokens`, `raw_drafts` |
| `[trim]` | execute 入口 | before/after schedule、每 req 的 `raw_spec`, `kept_spec`, `new_sched` |
| `[target_prepare]` | forward 前 | `num_draft_tokens`, `logits_indices`, `draft_token_ids` |
| `[target_batch]` | batch 布局 | `num_scheduled`, padding, `uniform_decode` |
| `[target_forward]` | forward 后 | `input_ids`, `positions`, **`sample_hidden_nan`**, `logits_indices` |
| `[verify]` | rejection 后 | `accepted_tokens`, `raw_sampled`, `spec_num_draft` |

## 多并发排查 Checklist

### 1. propose 是否正常？

```
[prune] kept=[3, 2]          # 各 req 保留数合理
[draft_result] echo_draft_tokens={'0-xxx': [tok,...], '1-yyy': [...]}
```

- `kept` 之和应 ≤ `k_max`
- `echo_draft_tokens` 不应为空（除非该 req 被完全剪枝）

### 2. trim 是否写入有效 spec？

```
[trim] kept_spec=[2164, 198, 90]   # ✅ 真实 token
[trim] kept_spec=[]                # ❌ 仍有问题
[trim] raw_spec=[-1, -1, ...]      # scheduler placeholder，正常但 trim 应用 echo_draft_tokens 覆盖
```

### 3. target forward 输入是否正确？

```
[target_forward] input_ids=[763, 328, 100540, ...]   # ✅ 无 trailing 0
[target_forward] input_ids=[..., 0, 0]               # ❌ scatter/trim 仍有问题
[target_forward] sample_hidden_nan={'0-xxx': False}  # ✅
[target_forward] sample_hidden_nan={'1-yyy': True}   # ❌ 该 req hidden 已损坏
```

### 4. verify 输出是否合理？

```
[verify] raw_sampled=[[71093, 198, ...], [4321, ...]]   # ✅ 非 0
[verify] raw_sampled=[[0, -1, -1, ...], ...]             # ❌ 采样失败
[verify] spec_num_draft=[5, 3]                             # 与 trim 一致
[verify] spec_num_draft=None                               # 可能未进入 spec decode 路径
```

## 典型故障模式

| 日志特征 | 可能原因 |
|----------|----------|
| `kept_spec=[]` | scheduler `-1` pad，未用 `echo_draft_tokens` |
| `sample_hidden_nan=True` + `input_ids` 含 0 | `_prepare_input_ids` 未正确写入 |
| req1 正常 req2 NaN | req_id 错配（检查 dict key 是否为正确 req_id） |
| `cum_log_probs` 含 NaN | 上游 forward 已损坏，先查 `input_ids` |
| `AssertionError` num_reqs_padded | 勿动态改 `uniform_decode_query_len` |

## `[prune]` 非单调 WARNING

若出现 `cum_log_probs not monotonic`，说明数值异常（通常伴随 NaN/inf），应回溯 `[target_forward]` 的 `input_ids` 与 `sample_hidden_nan`。
