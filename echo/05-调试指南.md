## 开启调试

```bash
export VLLM_ECHO_DEBUG=1
python your_script.py 2>&1 | grep "ECHO \["
```

## 日志阶段

| 前缀 | 位置 | 关键字段 |
|------|------|----------|
| `[draft_propose]` | propose 前 | `echo_k_max`, `batch_size`, `draft_step`, `req_ids` |
| `[prune]` | Eagle 剪枝 | `cum_log_probs`, `mask`, `kept`, `draft_before/after` |
| `[draft_result]` | propose 后 | `echo_cu_draft_tokens`, `echo_draft_tokens` |
| `[trim]` | `_update_states` 内 | before/after `total`、`schedule`、`old_sched`/`new_sched` |
| `[target_prepare]` | forward 前 | `num_draft_tokens`, `logits_indices`, `draft_token_ids` |
| `[target_batch]` | batch 布局 | `num_scheduled`, **`num_computed_tokens`** |
| `[target_forward]` | forward 后 | `input_ids`, `positions`, **`sample_hidden_nan`** |
| `[verify]` | rejection 后 | `raw_sampled`, `spec_num_draft` |

## 多并发排查 Checklist

### 1. trim 是否正确？

```
[trim] after total=2679 schedule={'0-xxx': 6, '1-yyy': 2673}
```

- `total` **必须等于** 各 req `new_sched` 之和
- prefill req 的 `new_sched` 应等于 prompt 长度（如 2673），**不是 1**
- 仅出现在 `details` 里的 req 才是被 trim 的 decode req

### 2. num_computed 是否合理？

```
[target_batch] num_computed_tokens=[2681, 2673]
```

- mixed decode 第一步：prefill req 应为 `prompt_len`（2673），decode req 为已累计值
- **异常**：solo req 出现 `prompt_len + 8`（如 2681 = 2673+8）→ 过计数未修正
- **异常**：`positions` 为大负数（如 -1756）→ 检查是否误改 `utils.py` participating 逻辑

### 3. target forward 输入

```
[target_forward] input_ids=[58, 198, 262, 328, 71093, 2164, 198]  # ✅
[target_forward] input_ids=[0]                                       # ❌
[target_forward] sample_hidden_nan={'1-yyy': False}                  # ✅
```

### 4. verify 输出

```
[verify] raw_sampled=[[198, 262, 313, -1], [71093, -1, ...]]  # ✅
[verify] raw_sampled=[[...], [0, -1, -1, -1]]                  # ❌
```

- `accepted_tokens={}` 在 async scheduling 下**正常**
- `spec_num_draft` 应与 trim 后 draft 数一致

## 典型故障模式

| 日志特征 | 可能原因 | 对应 Bug |
|----------|----------|----------|
| prefill `new_sched=1` | trim 未 skip prefill | #6 |
| `ValueError broadcast (2679,) vs (6,)` | `total` 未 sum 全 req | #6 |
| `num_computed=2673+8` | scheduler 乐观过计数 | #7 |
| `positions=-1756` | 错误的 participating 放宽 | #9 |
| `input_ids=[0]` after echo_keep=0 | sample 未写入 | #8 |
| `kept_spec=[]` | 未用 `echo_draft_tokens` | #3 |
| `sample_hidden_nan=True` | input_ids/position 错误 | #4/#7 |

## 推荐日志抓取命令

完整链路（单次 engine step）：

```bash
grep -E "ECHO \[(trim|target_prepare|target_batch|target_forward|verify|draft_result)\]"
```
