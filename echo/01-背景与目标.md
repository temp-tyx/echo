# 01 — 背景与目标

[← 返回目录](./README.md)

## 什么是 ECHO

ECHO（Enhanced Concurrency with High-confidence prOuning）是一种面向高并发的 speculative decoding 优化：

1. Draft model（如 Qwen3.5-4B MTP3）先 propose 多步 draft token
2. 计算每步条件 log-prob，累加得到 `cum_log_probs`
3. 在 **整个 batch 的所有 (request, step) 单元格** 上做 global top-k，只保留置信度最高的 `K_max` 个
4. 被剪枝的位置填 `-1`，后续 target model 不再验证这些 draft

目标：在 batch size 较大时，把有限的 compute budget 集中在高置信 draft 上，提高有效吞吐。

## 测试场景

| 场景 | 状态 |
|------|------|
| 单并发（batch_size=1） | 精度正常 |
| 多并发（batch_size>1） | 曾出现乱码、NaN、输出 token 为 0；经多轮修复后仍在排查 |

模型：**Qwen3.5-4B MTP3**（Eagle/MTP speculative decoding）

## 环境变量

定义于 `vllm-ascend/vllm_ascend/envs.py`：

| 变量 | 默认值 | 含义 |
|------|--------|------|
| `VLLM_ECHO_ENABLED` | `1` | 是否启用 ECHO |
| `VLLM_ECHO_K_MAX` | `5` | global top-k 保留的最大 draft 单元格数 |
| `VLLM_ECHO_STEPS_MULTIPLIER` | `2` | draft 步数扩展倍数，`draft_step = min(max(multiplier * k_max // batch_size, 1), max_spec_num)` |
| `VLLM_ECHO_MAX_SPEC_NUM` | `7` | draft 最大步数，用于 buffer 预分配 |
| `VLLM_ECHO_DEBUG` | `0` | 是否输出 `ECHO [...]` 诊断日志 |

## draft_step 动态计算

在 `propose_draft_token_ids` 中，每次 propose 前根据 batch size 调整实际 draft 步数：

```python
echo_k_max = envs.VLLM_ECHO_K_MAX
batch_size = spec_decode_common_attn_metadata.batch_size()
draft_step = min(
    max(envs.VLLM_ECHO_STEPS_MULTIPLIER * echo_k_max // batch_size, 1),
    self.drafter.num_speculative_tokens,
)
```

示例：`k_max=5, multiplier=2, batch_size=2` → `draft_step = min(max(10//2, 1), 7) = 5`

## cum_log_probs 的 prefix 性质

对单个 request，每步 log-prob ≤ 0，因此：

```
S[r,0] ≥ S[r,1] ≥ S[r,2] ≥ ...
```

global top-k 若选中 `(r, j)`，则 `(r, 0..j-1)` 必然也在 top-k 中。  
**不会出现**「中间 step 保留、前面 step 被剪掉」的空洞 mask；每个 request 的 mask 一定是 `{0,1,...,t}` 前缀。
