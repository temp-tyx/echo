# 01 - 背景与目标

## ECHO是什么

ECHO = **E**nhanced **C**oncurrency with **H**igh-confidence pr**O**uning

一种针对vLLM-Ascend的speculative decoding优化策略。

## 核心思想

传统EAGLE3的局限：
- 每个请求独立产生draft tokens
- 不区分请求间的置信度差异

ECHO改进：
- **Global Top-K策略**: 跨所有请求选择条件概率最高的k个draft tokens
- 优势: 高置信度请求获得更多budget，低置信度请求自动减少
- 公式: `cum_log_probs[t] = log p(t1) + log p(t2|t1) + ... + log p(t_{t+1}|...)`

## 环境变量配置

```bash
export VLLM_ECHO_ENABLED=1          # 开启ECHO
export VLLM_ECHO_K_MAX=5            # 全局最多保留5个draft tokens
export VLLM_ECHO_STEPS_MULTIPLIER=2 # draft_step = max(1, 2*5//batch_size)
export VLLM_ECHO_MAX_SPEC_NUM=7     # draft buffer最大7步
export VLLM_ECHO_DEBUG=1            # 开启调试日志
```

## 目标场景

- Qwen3.5-4B + MTP3
- 2请求并发：prefill(~2673 tokens) + decode
- async scheduling + spec decode

## 预期行为

| Step | Batch | req0 | req1 | 说明 |
|------|-------|------|------|------|
| 1 | [req0 prefill] | prefill 2673 | - | 正常 |
| 2 | [req0 decode, req1 prefill] | decode 6 tok | prefill 2673 | 正常 |
| 3 | [req0 decode, req1 decode] | decode 4 tok | decode 3 tok | **当前卡在这里** |
| 4+ | [req1 solo] | - | decode | req0完成后req1单独decode |
